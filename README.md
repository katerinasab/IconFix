# Icon Token Repair

Figma-плагин, чинящий привязку заливки/обводки у иконок, которую Figma иногда
сбрасывает на общий плейсхолдер `current-color` (переменная `current-color`
из удалённой коллекции `defaultColors`) — вместо настоящего семантического
токена компонента.

## Что делает

Запускается на текущем выделении (выделите нужные шейпы/инстансы и запустите
плагин — UI-панели нет, всё сразу выполняется и закрывается):

1. Спускается вглубь выделения, включая инстансы, и находит все `fills`/`strokes`,
   привязанные к `current-color`/`defaultColors` — как полностью потерянные
   привязки, так и случаи, когда «мусорный» paint добавился поверх/рядом с
   уже корректным.
2. Для каждой найденной иконки поднимается вверх по дереву до ближайшего
   "настоящего" компонента (пропуская тонкие обёртки вроде `icon-wrapper`),
   определяет имя компонента и состояние (`State`/аналог) и ищет подходящий
   токен по конвенции `<component>/color/.../<state>` — сначала в локальных
   коллекциях (`Theme` и др.), затем среди переменных подключённых
   shared/team-библиотек.
3. Если находится ровно один явно лучший кандидат — привязывает его
   автоматически. Если кандидатов ноль или несколько с одинаковым рейтингом —
   не гадает: такие ноды остаются выделенными на холсте, а варианты-кандидаты
   выводятся в консоль (`Plugins → Development → Show/Hide console`) для
   ручного разбора.
4. По завершении показывает `figma.notify` со сводкой (`fixed N, needs review M`).

Настраиваемые вещи вынесены в [`config.ts`](config.ts): список
placeholder-переменных/коллекций, список "тонких" обёрток и имён
variant-свойств, отвечающих за состояние.

## Разработка

```
npm install
npm run build     # tsc --noEmit (тайпчек) + esbuild-бандл code.ts -> code.js
npm run watch      # то же самое, но пересобирает при изменениях
npm run lint
```

Исходники разбиты на модули (`code.ts`, `traversal.ts`, `matching.ts`,
`binding.ts`, `config.ts`, `types.ts`) и бандлятся esbuild'ом в единый
`code.js` (см. [`build.mjs`](build.mjs)) — Figma грузит только один файл,
поэтому `import`/`export` между исходниками работают лишь благодаря бандлингу.

Плагин уже зарегистрирован в Figma Desktop (Plugins → Development → IconFix) —
после `npm run build` просто перезапустите его в Figma, повторный импорт
манифеста не нужен.
